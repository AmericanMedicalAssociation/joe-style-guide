Working with our style guide

Structure

This living style guide is built with ../link.twig and adheres to the principles of ../link.twig.

Issues

If you have questions or encounter an issue working with the style guide, report it as ../link.twig.

Contributions

For any code that you would like to contribute to the style guide, first create an issue in Github explaining clearly what problem you are trying to solve.

Make sure you have adhered to the guidelines outlined in the ../link.twig before submitting a pull request. Reference the Github issue you created when you submit your pull request.

Annotation and Documentation

Each pattern should have its own accompanying ../link.twig file that contains 1) a useful description of the pattern and 2) link to the relevant Jira tickets/Github issues relating to that pattern's creation. This information is needed to provide context as to how and when a particular pattern should be used.
